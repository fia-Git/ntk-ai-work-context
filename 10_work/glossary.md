# Glossary

このファイルは、NTKCuttingToolsでのヘルプデスク業務、問い合わせ対応、資料作成、上司向け報告、ユーザー向け案内文作成で使用する用語を整理するための用語集です。

AIは、このファイルを参照して、業務用語や略語の意味を理解し、必要に応じてユーザー向け・上司向けに分かりやすく言い換えてください。

---

## 基本方針

AIは、用語を扱う際に以下を守ってください。

- 不明な用語は勝手に断定しない
- 一般的な意味でよい用語は、一般的なIT用語として扱う
- NTKCuttingTools内での使われ方が書かれている用語は、その文脈を優先する
- ユーザー向けには専門用語をなるべく分かりやすく言い換える
- 上司向けには、必要に応じて用語の意味や業務影響を補足する
- 社外向け文章では、社内用語をそのまま使わず、必要に応じて説明を加える

# 1. 社内・業務管理系の用語

| 用語 | 意味 | 業務上の使われ方・補足 |
|---|---|---|
| バックログ | 問い合わせや作業を課題として登録・管理するシステム | 問い合わせ受付、作業管理、進捗管理、対応履歴の記録に使用する |
| 定型業務一覧 | 誰がどの業務を担当するか確認するExcelファイル | 担当者の確認や、バックログ登録時の担当者判断に使用する |
| 保守対応 | 既存システムや既存業務を安定運用するための対応 | バックログの業務区分の一つ |
| 運用業務 | 日常的に発生する定型的・継続的な業務 | バックログの業務区分の一つ。手順書修正や定型作業などで使用する |
| 問い合わせ | ユーザーや関係者からの質問・確認依頼 | バックログの業務区分の一つ |
| 障害 | システムや端末などに問題が発生し、業務に影響が出ている状態 | バックログの業務区分の一つ。影響範囲や緊急度を明確にする必要がある |
| 作業計画書 | 作業の目的、対象、手順、影響範囲、切り戻し手順などを整理した資料 | サーバー作業、設定変更、影響がある作業などで作成する |
| WBS | 作業を細かいタスクに分解し、担当者・期限・状態を管理する表 | 複数工程がある作業や、進捗管理が必要な案件で使用する |
| 業務フロー | 業務の流れを図や手順で整理したもの | 誰が、いつ、何をするかを明確にするために作成する |
| 手順書 | 作業手順を順番にまとめた資料 | ユーザー、外注、ヘルプデスクメンバー、技術者向けに作成する |

## バックログ登録時によく使う項目

| 項目 | 意味・補足 |
|---|---|
| 課題名（連絡者） | バックログに登録する課題のタイトル。連絡者や対象内容を分かるようにする |
| 端末名/PC/Smartphone name | 対象となるPC、スマートフォン、端末名 |
| 誰が/Who | 実際に影響が出ている人 |
| 何を/What | 何が起きているか、何を依頼されているか |
| なぜ/Why | なぜ対応が必要か、依頼の目的は何か |
| どうなった/result | 現在どうなっているか、結果として何が起きているか |
| 業務影響/business impact | 業務にどのような影響があるか |
| いつからどのくらい/Since when and for how long | いつから発生しているか、どのくらい継続しているか |
| 次にやる事/next action | 次に行う作業 |
| 達成基準/GOAL | 何をもって完了とするか |
| 残作業/ToDo | まだ残っている作業 |

# 3. セキュリティ・端末管理系の用語

| 用語 | 意味 | 業務上の使われ方・補足 |
|---|---|---|
| MaaS360 | MDMとして使用しているスマートフォン管理ツール | 主にスマートフォン管理で使用する。MDM上からユーザーのスマホのパスワードリセットやポリシー変更を行う |
| MDM | Mobile Device Managementの略。スマートフォンなどのモバイル端末を管理する仕組み | NTKCuttingToolsでは主にスマホ管理の意味で使用する |
| Trellix | PC起動時にログイン画面が表示される暗号化関連の仕組みとして扱う | Trellixの配布、パスワード変更、ログイン関連の問い合わせで使用する |
| MDE | Trellixに関連する暗号化のこととして扱う | PC起動時の認証や暗号化関連の文脈で使用する |
| Palo Alto | ネットワーク管理・セキュリティ関連の製品・仕組み | 一般的な意味で扱う。ネットワークやファイアウォール関連の問い合わせで出てくる |
| フィッシングメール | 偽メールにより情報を盗もうとする攻撃メール | ユーザーから不審メールの報告があった場合に対応する |
| FWの穴あけ | ファイアウォールで必要な通信を許可する設定変更 | 通信許可、接続要件、セキュリティ確認の文脈で使用する |

---

## セキュリティ関連でAIが注意すること

セキュリティ関連の問い合わせを整理する場合、AIは以下を意識してください。

- 影響範囲を確認する
- 緊急度を確認する
- 誰に影響が出ているかを確認する
- 業務影響を確認する
- パスワードや認証情報は記録しない
- FWの穴あけなどは、目的・接続先・必要な通信・リスクを整理する
- フィッシングメールは、送信元、件名、本文、添付ファイル、URL有無を確認する
- 不明点は推測で断定しない

| 用語 | 意味 | 業務上の使われ方・補足 |

## CAD・設計系でAIが注意すること

CADや設計系の問い合わせは、ユーザーの業務や専門システムに関わる場合があります。

AIは以下を意識してください。

- 一般的な意味で分かる範囲と、社内運用の違いを分ける
- インストール方法、使い方、サーバー、権限、接続先などを整理する
- 不明点は勝手に断定せず確認する
- ユーザー向けには専門用語を必要に応じて噛み砕く
- 上司向けには、業務影響や対応方針を整理する

# 6. IMC社・海外問い合わせ関連

| 用語 | 意味 | 業務上の使われ方・補足 |
|---|---|---|
| IMC社 | イスラエルにあるNTKの親会社 | 必要に応じて英語で問い合わせる |
| 英語問い合わせ | IMC社に対して技術的な確認や依頼を行うこと | 技術的な確認、依頼、調査、回答内容の整理などで使用する |

---

## IMC社へ問い合わせる時にAIが意識すること

IMC社へ英語で問い合わせる場合、AIは以下を意識してください。

- 日本語で内容を整理してから英語文を作成する
- 問い合わせの目的を明確にする
- 現在の状況を簡潔に説明する
- 何を確認したいのかを明確にする
- 必要な情報を箇条書きで整理する
- 丁寧だが、回りくどすぎない英語にする
- 技術的な用語は誤訳しないよう注意する
- 不明点は「要確認」とする

## パソ決関連でAIが注意すること

パソ決関連の問い合わせや資料作成では、AIは以下を意識してください。

- パソ決はパソコン決裁システムのこととして扱う

- 検収と納品報告のように抜けている手順がある場合は、作業漏れや認識違いのリスクを整理する

# 10. 上司向けに補足する時の例

上司向けには、単に用語を説明するだけでなく、業務影響や判断材料も含めてください。

## 例：FWの穴あけ

```text
〇〇のシステムを利用するために、特定の通信を許可する必要があります。
いわゆるFWの穴あけ対応になりますが、通信先や用途を確認したうえで進める必要があります。
目的は〇〇で、許可しない場合は〇〇が利用できない可能性があります。
```

## 例：MaaS360

```text
対象スマートフォンの管理はMaaS360上で行う必要があります。
MaaS360からパスワードリセットやポリシー変更ができるため、今回の対応ではまずMDM上の状態を確認します。
```

## 例：Trellix

```text
対象PCは起動時にTrellixの認証画面が表示される状態です。
そのため、通常のWindowsログイン前にTrellix側のパスワード確認・変更が必要になる可能性があります。
```

## 例：パソ決

```text
パソ決はパソコン決裁システムのことで、今回の件ではID購入や印影購入の手順が関係します。
現行手順では検収と納品報告の手順が不足しているため、業務フローと手順書の整合性を見直す必要があります。
```

# 11. AIが用語集を使う時のルール

AIは、この用語集を以下の目的で使用してください。

- ユーザー向けに分かりやすく言い換える
- 上司向け説明で用語の意味を補足する
- 手順書やフロー作成時に用語の意味を揃える
- 問い合わせ内容を整理する時に、対象システムや業務領域を理解する
- バックログ登録文のカテゴリー判断に活用する
- 不明な用語は勝手に断定せず、ユーザーに確認する

---

# 12. 不明な用語が出てきた場合

AIは、不明な用語が出てきた場合、勝手に意味を決めつけないでください。

以下のように確認してください。

```text
「〇〇」の意味がこのリポジトリ内では確認できませんでした。
一般的には〇〇を指す可能性がありますが、NTKCuttingTools内での意味が異なる可能性があるため、社内での意味を確認させてください。
```

または、

```text
この用語は詳細が不明なため、現時点では「要確認」として扱います。
```

# 14. 注意事項

この用語集は、AIが業務文脈を理解するための補助資料です。

用語の意味が一般的な意味と社内での意味で異なる場合は、社内での使われ方を優先してください。  
ただし、詳細が不明な場合は、推測で断定せず「要確認」としてください。
